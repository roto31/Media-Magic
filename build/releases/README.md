# Media Magic — preview releases index

All builds listed here are **preview / prerelease** quality: suitable for testing, not warranted for production. Each corresponds to a SemVer **build ID** `0.1.0+N` where `N` is the monotonic **BUILD_NUMBER** (see `BUILD_NUMBER` at repo root on `main`).

## Tag convention

Git annotated tags use the form **`v0.1.0+N-preview`** (example: `v0.1.0+7-preview`). Each tag’s message states the preview scope. **GitHub Releases:** enable **“Set as a pre-release”** when publishing assets.

## Artifact layout (when `builds/` is present)

```
builds/<BUILD_ID>/MediaVault.app
builds/<BUILD_ID>/MediaVault-<BUILD_ID>-macOS.zip   # when generated by release build
```

> **Note:** The app bundle currently uses the **MediaVault** executable/name in Info.plist on shipped builds; rebranding to **Media Magic** may appear in a future build. Release docs refer to the product as **Media Magic** where marketing applies; paths reflect on-disk names.
