# Build Process Documentation

This document describes the current build pipeline implemented by `build.sh`.

## Modes

- `./build.sh`:
  - Debug/local build only.
- `./build.sh release`:
  - Production-oriented build with signed-only GitHub distribution flow.
- `./build.sh release sign`:
  - Compatibility alias; same release behavior.

## Version Inputs

- `VERSION`:
  - SemVer core (`MAJOR.MINOR.PATCH`), currently Alpha baseline (`0.1.0`).
- `BUILD_NUMBER`:
  - Numeric counter incremented after each successful build.

Build ID format:
- `<VERSION>+<BUILD_NUMBER>`
- Example: `0.1.0+7`

## Build Artifact Layout

For each successful build:

- App bundle:
  - `builds/<BUILD_ID>/MediaVault.app`
- Release zip:
  - `builds/<BUILD_ID>/MediaVault-<BUILD_ID>-macOS.zip`

## End-To-End Build Flow

```mermaid
flowchart TD
    Begin["build.sh invoked"] --> DetectMode{"release mode?"}
    DetectMode -->|"No"| DebugPath["Compile + write app bundle"]
    DetectMode -->|"Yes"| ReleasePath["Compile + write app bundle"]
    DebugPath --> DebugDone["Done"]
    ReleasePath --> Sign["Developer ID sign app"]
    Sign --> Verify["codesign verify"]
    Verify --> Zip["Create release zip asset"]
    Zip --> GhChecks["Validate gh auth + git repo context"]
    GhChecks --> ReleaseCheck{"Release tag exists?"}
    ReleaseCheck -->|"No"| Create["gh release create"]
    ReleaseCheck -->|"Yes"| Update["gh release edit notes"]
    Create --> Upload["gh release upload --clobber"]
    Update --> Upload
    Upload --> ReleaseDone["Done"]
```

## CI/CD-Style Release Actions (Local Script)

```mermaid
sequenceDiagram
    participant Dev as Developer
    participant Script as build.sh
    participant GH as GitHub Releases

    Dev->>Script: ./build.sh release
    Script->>Script: Validate VERSION + BUILD_NUMBER
    Script->>Script: Build app bundle
    Script->>Script: Sign and verify app
    Script->>Script: Package zip asset
    Script->>GH: Create or update release metadata
    Script->>GH: Upload artifact asset
    GH-->>Dev: Release contains latest zip
```

## Signing/Distribution Policy

- Distribution mode is **signed-only** (Developer ID).
- Notarization is intentionally out-of-scope at the moment.
- Users may need first-launch Gatekeeper bypass on downloaded artifacts:
  - Right-click app -> Open
  - Or remove quarantine:
    - `xattr -dr com.apple.quarantine MediaVault.app`

## Release Notes Automation

Release notes are generated by script and include:
- version/build identifiers
- artifact filename
- signed/not-notarized distribution note
- Gatekeeper workaround guidance
